Day 30-60

Exploration

Transitioning from foundational knowledge to deep structural thinking. This phase encourages engineers to question, debate, and architect robust solutions.

Phase Description

The Exploration phase is all about critical thinking and system design. Engineers engage in complex architectural discussions and debates. They learn to evaluate trade-offs, understand microservices boundaries, and design scalable platforms that meet enterprise requirements.

Key Technologies

  • System Design: Architectural patterns and anti-patterns.
  • Cloud Architecture: Multi-region deployments and high availability.
  • Containerization: Docker and Kubernetes exploration.

Learning Journey

  • Deep engineering debates & peer reviews
  • Architecture whiteboarding sessions
  • Platform concepts and domain-driven design

Engineering Debates

Engineering Debate 1

Google ADK vs LangChain

Google’s Agent Developer Kit (ADK) is a framework optimized for building agents within Google’s ecosystem. LangChain is an open‑source framework for building LLM apps and agents with broad model, tool, and infrastructure flexibility.

Engineering Debate 2

CrewAI vs LangChain

CrewAI is focused on multi-agent collaboration, defining agent roles, tasks, and teamwork patterns. LangChain focuses on building pipelines, retrieval, and tools, with optional agent support.

Engineering Debate 3

Microsoft Agent Framework vs LangChain

The Microsoft Agent Framework provides a structured way to build orchestrated AI agents integrated with Microsoft 365, Azure OpenAI, and enterprise data. LangChain remains broader and more tool-based.

Engineering Debate 4

Strands Agent vs LangChain

Strands is Microsoft’s new orchestrated AI Agent system built on top of the Foundry architecture. It focuses on autonomous task decomposition and orchestration. LangChain focuses more on tool-wrapping and chaining.

Engineering Debate 5

Weaviate vs MongoDB Atlas Vector Search

Weaviate is a vector-native database designed for semantic search and AI apps. MongoDB Atlas adds vector search features to a general document database.

Engineering Debate 6

Python vs JavaScript/TypeScript for AI Apps

Python dominates in model development, prototyping, and data work. JavaScript/TypeScript dominate frontend, full‑stack, and edge AI.

Engineering Debate 7

Apache Iceberg vs Delta Lake

Iceberg and Delta Lake are open-table formats for data lakes. Iceberg is vendor-neutral, while Delta Lake is Databricks-optimized but now open source.

Engineering Debate 8

Pulumi vs Terraform for AI Infrastructure

Terraform uses declarative HCL; Pulumi uses real programming languages (Python, TS, Go) for IaC.

Engineering Debate 9

Microservices vs Event-Driven Architecture

Microservices break the system into small services with APIs. Event-driven architecture connects components through asynchronous events.

Engineering Debate 10

Microsoft Foundry: Platform vs Lock-in

Microsoft Foundry is a platform for building enterprise AI apps with orchestration, safety, grounding, and governance. The debate is between deep integration advantages vs vendor lock‑in risks.

Engineering Debate 11

REST API vs GraphQL for AI Data Access

REST uses resource-based endpoints; GraphQL allows querying exactly the fields you want.

Outcome

At the end of day 60, our engineers are no longer just writing code. They are designing scalable, fault-tolerant systems and can confidently present and defend their architectural decisions.
